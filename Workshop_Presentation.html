What is AI?

A clear, honest explanation — from the ground up.

The Simple Definition

Artificial Intelligence is a system that predicts the next best output based on patterns learned from large amounts of data.


It does not think. It does not understand. It predicts — extremely well, at enormous scale.

Traditional Programming
A programmer writes exact rules.
The computer follows rules precisely.

if input == "hello" → reply "hi"

If the rule does not exist — the program fails.
Artificial Intelligence
We give millions of examples.
AI finds patterns by itself.

learns from billions of examples → predicts

It can handle inputs it has never seen before.

What is Generative AI?

Generative AI is a specific type of AI that creates new content — text, code, images, audio — rather than just classifying or searching.


  • ChatGPT, Claude, Gemini → generate text and code
  • DALL·E, Midjourney → generate images
  • GitHub Copilot → generate and complete code

In this workshop we focus on text and code generation — the most useful for IT students.

DAY 1 · 3 of 14

How ChatGPT-Style Models Work

Simple intuition — no math, no jargon. Just what you need to understand why prompts matter.

The Basic Flow

Your Prompt (Input)
→
AI Model (Prediction)
→
Response (Output)

The model was trained on billions of text documents — books, websites, code, conversations. During training it learned patterns: which words usually follow which other words, in which contexts.


When you send a prompt, the model predicts the most statistically likely helpful continuation based on everything it learned.

Training vs Using the Model

Training (done once, by the company)
Model reads billions of text examples.
Adjusts its internal weights to improve predictions.
Takes weeks on thousands of computers.

You do NOT control this.
Inference (what you do every day)
You send a prompt.
Model uses its trained weights to predict the best response.
Happens in seconds.

Your prompt IS your control lever.

Why the Same Question Can Give Different Answers

AI models add slight randomness to their output so they do not always repeat the same sentences. This is called temperature.


  • Low temperature → more predictable, consistent answers (good for code)
  • High temperature → more creative, varied answers (good for writing)

This is why your prompt needs to be precise — a vague prompt + randomness = unpredictable output.

Key Takeaway

The model is a very powerful pattern predictor. The quality of its output depends almost entirely on the quality of your input — your prompt. This is why prompt engineering exists.

DAY 1 · 4 of 14

AI Limitation: Hallucination

The most dangerous AI problem — and why every IT student must know about it.

What is Hallucination?

AI can generate responses that sound completely correct, professional, and confident — but are entirely false or invented. The model does not know it is wrong. It is simply predicting what a plausible-sounding answer looks like.

Example 1 — Fake Code (NASA API)

Prompt given to AI:

# Give me the Python function from the official NASA Mars Rover API to fetch photos

AI response (sounds completely real — but is entirely invented):

def get_mars_photos(rover_name: str, date: str) -> list: # This function fetches Mars rover photos from NASA return nasa_api.fetch_photos(rover=rover_name, earth_date=date)
What is wrong with this response:
  • The function nasa_api.fetch_photos() does NOT exist — never has
  • NASA's real API works through HTTP requests to a URL, not a Python object
  • If a student copies this and runs it — it will crash immediately
  • AI made it up because it "sounds like" what a NASA Python function might look like

Example 2 — Fake Facts

Asking AI about a made-up function name:

# What does the C function strsplit() do?

AI might confidently explain a function that does not exist in the C standard library — giving it fake parameters, a fake return value, and fake examples. All sounding perfectly correct.

How to Protect Yourself

  • Never run AI code without reading it first. Read every line. If a function name looks unfamiliar — search it.
  • Cross-check facts. If AI tells you something important — verify it against official documentation or a trusted source.
  • Ask AI to explain its answer. If the explanation has gaps or circular reasoning — it may be hallucinating.
  • Test the code. Always compile and run AI-generated code to see if it actually works.
DAY 1 · 5 of 14

AI Limitations: Bias & Errors

Two more important limitations every IT student should be aware of.

What is AI Bias?

AI models learn from human-written data. Human writing contains cultural assumptions, stereotypes, and historical biases. The model learns these patterns and can reproduce them in its output — without knowing it is doing so.

Bias Example — The Software Engineer Story

Prompt given to AI:

# Write a short story about a software engineer.

A typical AI response:

"John is a brilliant software engineer who leads a team of developers. He works long hours and loves solving hard math problems. His wife, a schoolteacher, always supports him from home."

What is biased here:
  • AI assumed the engineer is male — without being told
  • AI assumed the spouse is female in a domestic support role
  • This reflects historical gender stereotypes in training data
  • The model did not "decide" to be biased — it learned this pattern from millions of examples

Why it matters for you: When you use AI for research, reports, or explanations — be critical. Ask yourself: whose perspective is this reflecting?

What are AI Errors (Logic & Code Errors)?

Even when AI is not hallucinating, it can produce code with subtle logic errors, use outdated syntax, miss edge cases, or produce code that compiles but gives wrong results.

Error Example — Off-by-One in C

AI-generated code that compiles but gives wrong output:

#include <stdio.h> int main() { int arr[] = {10, 20, 30, 40, 50}; for (int i = 0; i < 6; i++) { /* Bug: should be i < 5 */ printf("%d\n", arr[i]); /* reads beyond array boundary */ } }

This code will compile without errors but will read memory beyond the array — a classic error AI makes when it is not given the exact array size in the prompt.

Golden Rule for All Three Limitations

Never blindly copy AI output. Always read it, understand it, test it — then use it.

Why Prompt Engineering Matters

You already use AI. This section explains why learning to use it properly changes everything.

The Simple Truth

Bad Prompt
"Write about AI"
Vague. AI does not know: Which aspect of AI? What length? What audience? What format?

Result: generic, often useless output.
Good Prompt
"Write a 200-word explanation of how AI is used in healthcare to improve early disease detection. Include two real-world examples. Write for a beginner IT student."
Result: focused, accurate, useful output.

Five Reasons Prompt Engineering Matters

🎯
Precision
Well-crafted prompts reduce ambiguity and dramatically improve output quality
⚡
Efficiency
Good prompts mean fewer retries. You get the right answer faster.
💰
Cost-Effective
Better prompts use fewer AI tokens and fewer API calls. Saves money at scale.
🔁
Reliability
Consistent prompt patterns produce predictable, repeatable results.
🔒
Security
Proper prompt design helps prevent misuse, injection attacks, and unsafe outputs.

Think of It This Way

AI is an incredibly powerful engine. A bad prompt is like driving with your eyes closed — the engine works, but you will crash.


A good prompt is the steering wheel. Prompt engineering is the skill of steering.

DAY 1 · 7 of 14

Core Principle 1: Clarity & Specificity

Write prompts that leave absolutely no room for misinterpretation.

The Principle

Use specific language. Avoid ambiguous terms. Start with clear action verbs. Every word in your prompt should serve a purpose.

  • Use: Analyze, Summarize, Create, Explain, Fix, List, Compare
  • Avoid: "look at", "tell me about", "help with", "do something with"
❌ Not Specific
"Look at this and tell me about it"
What should AI look at? What aspect? What kind of response? There is no action verb, no scope, no target.
✅ Specific
"Analyze this C program and list the top 3 logic errors, with an explanation of each."
Action verb: Analyze + List. Scope: C program. Target: logic errors. Format: top 3 with explanations.

Programming-Specific Examples

❌ Vague C prompt: "Explain functions in C" ✅ Specific C prompt: "Explain C functions to a beginner: cover definition, syntax, parameters, return values, and show a complete working example that calculates the area of a rectangle." ❌ Vague Python prompt: "Tell me about loops" ✅ Specific Python prompt: "Compare for loops and while loops in Python. Show one example of each using a list of 5 numbers. Explain when to use each type."
Quick Rule: After writing your prompt, ask: "Could AI misunderstand what I want here?" If yes — make it more specific before sending.
DAY 1 · 8 of 14

Core Principle 2: Context Provision

AI does not know your situation unless you tell it. Context is everything.

The Principle

Provide sufficient background information so the model understands the scenario, who you are, what you already know, and what specifically you need. Without context, AI gives a generic answer. With context, it gives a targeted answer.

❌ No Context
"Fix this code"
AI does not know: What language? What is it supposed to do? What error is occurring? Where does it fail?
✅ With Context
"Fix this Python function that should calculate compound interest. The current error is a division by zero when the rate is 0. Here is the code: [code block]"
Language ✓ · Purpose ✓ · Error description ✓ · Code provided ✓

What Context to Include

  • Your level: "I am a first-year BIT student learning C for the first time"
  • What you already know: "I understand basic if/else but not recursion yet"
  • The goal: "I need to submit this as an assignment"
  • What is going wrong: "My program compiles but gives wrong output for negative numbers"
  • The environment: "I am using GCC on Windows / using VS Code / using an online compiler"

Context Makes the Difference

❌ No context: "Explain machine learning" → AI writes a textbook-level essay you cannot use ✅ With context: "Explain machine learning to a first-year BIT student who knows Python basics but has never studied AI before. Use a real-world analogy. Keep it under 200 words." → AI writes something you can actually understand and use

Core Principle 3: Task Decomposition

Break complex tasks into smaller steps — AI handles each step better.

The Principle

Instead of asking AI to do everything in one giant prompt, break the task into clear, numbered sub-tasks. This improves accuracy, makes the output more organised, and makes it easier for you to verify each part.

❌ All at Once
"Teach me C programming"
Too broad. AI does not know where to start, what depth to use, or when to stop. The output will be massive and disorganised.
✅ Decomposed
"Teach me arrays in C by doing the following steps:
1. Explain the concept
2. Show the syntax
3. Provide a working example
4. List 2 common beginner mistakes"
Clear steps. AI follows them in order. Each part is verifiable.

Real Decomposition Example — Dataset Analysis

❌ Overloaded: "Analyze this dataset and provide insights" ✅ Decomposed: "Please analyze this student grade dataset following these steps: 1. Summarize the data structure and key columns 2. Calculate the average, minimum, and maximum marks 3. Identify which subjects have the most failing students 4. Suggest 2 actionable improvements based on the data"

Programming Learning Example

Ask AI to teach you a concept in structured steps: "Explain C++ classes using this structure: 1. What is a class? (one paragraph, simple language) 2. Syntax of a class definition (with a real-world example) 3. A complete working C++ program using a class called 'Car' 4. What are the most common beginner mistakes with classes?"
Why it works: Each numbered step gives AI a clear deliverable. You can read each part, verify it, and continue — rather than getting an overwhelming wall of text.
DAY 1 · 10 of 14

Core Principle 4: Output Format Specification

Tell AI exactly how you want the answer structured — and it will deliver it that way.

The Principle

Clearly define the expected format, structure, and length of the response. If you need code + explanation, say so. If you want bullet points, ask for bullet points. If you need a specific length — specify it.

Format Options You Can Request

💬
Code + Comments
Ask for code with inline comments on every line — the best way to learn what each line does.
🔢
Step-by-Step
Numbered steps — ideal for algorithms, debugging procedures, and multi-part explanations.
📊
Table / Comparison
A side-by-side table — great for comparing C vs C++ vs Python syntax at a glance.
🗂️
JSON / Structured
Ask for JSON output — clean, structured, easy to read and use in your own code.
📏
Word Limit
"Explain in 3 sentences" or "under 100 words" — controls exactly how much AI writes.

Format Examples for Programming

✅ Code with explanation format: "Write a C program that finds the largest number in an array. For each line of code, add a comment explaining what it does. After the code, write a 3-sentence explanation of the logic." ✅ Comparison table format: "Create a comparison table of for loops in C, C++, and Python. Columns: Syntax, Example, Key Difference. Keep each cell under 2 lines." ✅ Structured output format: "Explain recursion in C using this format: - Definition (1 sentence) - When to use it (2 bullet points) - Working code example (with comments) - One common mistake beginners make"

Prompting Techniques

Different situations call for different techniques. Learn when to use each one.

1. Zero-Shot Prompting

Give a direct instruction with no examples. Works well for simple, clear tasks where the format is obvious.

Best for: simple tasks, translations, definitions "Translate this C variable declaration into Python syntax: int score = 0;" Result → Python: score = 0

2. Few-Shot Prompting

Show AI 2–3 examples of input → output pairs before asking your actual question. AI learns the pattern from your examples.

Example: teaching AI a custom format "Classify each statement as Good Prompt or Bad Prompt. Example 1: 'Write code' → Bad Prompt Example 2: 'Write a C function to reverse a string, with comments' → Good Prompt Example 3: 'Fix this' → Bad Prompt Now classify: 'Explain recursion in C with a factorial example for a beginner'"

Best practice: Use 2–5 examples. Keep them consistent in format. Never use more than 5 — it confuses the pattern.

3. Chain-of-Thought (CoT) Prompting

Ask AI to think step by step. This improves accuracy for complex tasks — especially debugging, math, and multi-step logic problems.

Instead of: "Why does this C program output the wrong answer?" Use: "Look at this C program step by step. First, trace through the code manually with input = 5. Then identify where the output diverges from what is expected. Finally, explain the bug."
Trigger phrase that works in most AI tools: "Let's think through this step by step." "Work through this systematically." "Trace the execution line by line."

4. Role-Based Prompting

Assign AI a specific role or persona. This guides the tone, depth, and style of the response to match what you need.

"You are a C programming tutor for complete beginners. Explain pointers as if your student has never seen them before. Use a simple real-world analogy first, then the syntax." vs. no role — which gives a generic, too-advanced explanation

Role-based prompting is especially powerful when you want the answer tailored to a specific skill level — which is exactly what you need as a student learning programming.

5. Negative Prompting

Explicitly tell AI what not to include. Use when you want to keep the response focused and avoid unnecessary complexity.

"Explain C pointers without using the words 'memory address' or 'hexadecimal'. Use only everyday analogies." "Write a Python sorting example. Do not use built-in sort() or sorted(). Implement bubble sort manually." "Explain what recursion is. Do not use Fibonacci — use a different example."

6. Iterative Refinement

Use AI as a conversation — build on its previous response. Each follow-up improves or extends what it gave you.

Turn 1: "Write a C function that finds the maximum value in an array." Turn 2 (refine): "Good — now add input validation. Handle the case where the array is empty or has only one element." Turn 3 (extend): "Now convert this to also track the index of the maximum value, not just the value itself."

This step-by-step approach lets you build complex solutions gradually, understanding each addition — rather than asking for everything at once and getting overwhelmed.

Common Prompting Pitfalls

The most frequent mistakes when writing prompts — and exactly how to fix each one.

Pitfall 1 — Ambiguous Instructions

Problem: Vague or unclear prompts lead to generic, inconsistent outputs. Using terms like "make it better", "analyze this", or "tell me about" without specific criteria leaves AI guessing.

❌ "Make this text better" ✅ "Rewrite this code comment to be more specific: 1. Use active voice 2. Mention the exact variable being modified 3. Keep it under 10 words"

Pitfall 2 — Overloading with Examples

Problem: Giving 8–10 examples in one prompt can confuse the AI. Too many examples, especially if they are inconsistent, dilute the pattern you are trying to establish.

❌ Providing 10+ examples with varying styles and formats ✅ Use 2–5 high-quality, consistent, clearly formatted examples that all follow the exact same pattern you want AI to replicate

Pitfall 3 — Ignoring Context Limits

Problem: Pasting your entire 500-line program and asking "fix everything" overloads the model. It will miss things, lose track of context, or give incomplete fixes.

Fix: Isolate the specific function or section that has the issue. Show only the relevant code. If your program is large — describe the structure and paste only the broken part.

Pitfall 4 — Inconsistent Formatting

Problem: Using a different prompt structure each time means you cannot build on what worked before. Random prompt styles produce random quality output.

Develop a consistent template you reuse: Role: [who AI should be] Task: [what you need] Context: [background, your level, the language] Output: [format, length, structure] Constraints: [what to avoid]

Pitfall 5 — Not Testing Edge Cases

Problem: Asking AI to generate code or explanations and only testing with the "ideal" scenario. Code that works for one input may fail on others.

Fix: After getting AI-generated code, always test with: the normal case, an empty/zero input, a very large input, and negative numbers (if applicable).

AI as a Tutor

How to use AI to understand programming concepts faster — with real examples in C, C++, and Python.

The Key Idea

Traditional learning: read the textbook → attend lecture → ask teacher (who has 30 other students).


With AI: ask for the exact explanation you need, at your level, with your choice of examples — instantly, as many times as you need, rephrased as many ways as you want.


AI = Personal Programming Tutor available 24/7.

C — Learning Concepts

Learning arrays from scratch: "Act as a C programming tutor. Explain arrays in C using these steps: 1. What is an array and why do we need it? 2. How to declare and initialise an array (syntax) 3. A complete working program: store 5 exam marks, find average 4. Two common beginner mistakes with arrays Audience: first-year BIT student. Add code comments." Learning pointers — scary topic, simplified: "Explain C pointers to a complete beginner. Start with a simple real-world analogy (not memory addresses yet). Then introduce the & and * operators with one simple example. Avoid malloc and dynamic memory for now."

C++ — Learning Object-Oriented Programming

Understanding classes: "You are a C++ tutor. Explain classes and objects using the analogy of a real car. Show: class definition, constructor, attributes, one method. Write a complete program where a Car object is created and its details are printed. Add a comment on every line." Comparing C and C++: "I know C already. Explain the 3 biggest differences between C and C++ that I will notice immediately when writing programs. Use side-by-side code examples."

Python — For Students Who Know C

Connecting what you know: "I know C programming. Teach me Python by comparing it to C. Cover: variables (no type declaration), for loops, functions, and lists vs arrays. For each concept show: C version → Python version → key difference." Understanding a confusing error: "I am getting this Python error: 'list index out of range'. Explain why this happens in simple terms. Show a minimal example that causes it. Show how to fix it and how to prevent it."

Code Generation with AI

How to use AI to generate working, readable, understandable code — not just code that runs.

❌ Lazy Generation Prompt
"Write a C program"
What program? What should it do? What data? What output format? What level?

AI will guess and produce something you probably cannot use or learn from.
✅ Good Generation Prompt
"Write a C program that takes 5 integer inputs from the user and prints the maximum and minimum values. Add a comment on each line. Include the expected output for inputs: 3, 7, 1, 9, 4."
Purpose ✓ · Input format ✓ · Expected output ✓ · Comments ✓ · Test case ✓

C — Generation Examples

Factorial with explanation: "Write a C program that calculates the factorial of a number using a for loop (not recursion). Add comments explaining each line. Show expected output for input = 6." Bubble sort — understanding the algorithm: "Write a C program implementing bubble sort on an array of 6 integers. After each pass of the outer loop, print the current state of the array so I can see how it sorts step by step. Explain each part of the algorithm in comments."

C++ — Generation Examples

Student class: "Write a C++ program with a class called Student. Attributes: name (string), rollNo (int), marks (float). Methods: a constructor, and a display() function that prints all attributes. In main(), create 2 Student objects with different data. Add comments on every line. No inheritance needed."

Python — Generation Examples

Function with error handling: "Write a Python function called calculate_average() that takes a list of numbers and returns the average. Handle the empty list case (do not divide by zero). Include a docstring. Show 3 test calls: normal, empty list, single item." Reading a file and processing data: "Write a Python program that reads numbers from a file called 'scores.txt' (one number per line), calculates the average, and prints the result. Handle the case where the file does not exist. Add comments on each important line."

⚠️ The Non-Negotiable Rule

Do not run AI-generated code without reading it first. Read every line. If you do not understand a line — ask AI: "Explain line 7 of this code in simple terms."


Code you do not understand is code you cannot fix, cannot explain to your teacher, and cannot learn from.

Debugging with AI

AI is an exceptional debugging assistant — if you give it the right information.

The Debugging Workflow

Find Error
→
Explain
→
Fix
→
Improve

Always ask AI to go through all four steps — not just give you the fixed code.

What to Include in a Debugging Prompt

  • The code — paste the full function or section that is broken
  • What it should do — the expected behaviour
  • What it actually does — the wrong output or error message
  • A test case — the input that causes the problem
  • What you want — explanation + fixed code + how to prevent the error

Debugging Prompt Structure

"Find the error in this C program. Explain step by step why the error happens. Show the corrected version. Tell me how to avoid this type of error in future. [paste your code here]"

Live Example — Off-by-One Bug

/* Should print numbers 1 to 5, but prints 1 to 4 */ #include <stdio.h> int main() { int i = 1; while (i < 5) { /* Bug: missing the = */ printf("%d\n", i); i++; } return 0; }
# Debugging prompt for this code: "This C while loop should print 1, 2, 3, 4, 5 but it only prints 1, 2, 3, 4. Find the bug, explain why it happens step by step, and show the corrected loop."

Debugging Error Messages

# You got a compiler error? Template-Based Prompting

Build a personal prompt library — reusable templates that produce consistent results every time.

Why Templates?

Writing a great prompt from scratch every time is slow and inconsistent. Templates let you fill in the variables for each new task while keeping the structure that you already know works.


A prompt template is a repeatable pattern for a specific type of task.

The Master Template Structure

Role: [who AI should be] Task: [what you need done, decomposed if complex] Context: [your level, language, relevant background] Output: [format, length, structure you want] Avoid: [topics, jargon, approaches to exclude]

Template 1 — Learning a New Concept

Role: Act as a [C / C++ / Python] tutor for beginners Task: Teach me [concept name] using: 1. Simple explanation (1 paragraph) 2. Syntax with annotations 3. A complete working program example 4. Two common mistakes beginners make Context: I am a first-year BIT student. I [know / do not know] [prior concept] Output: Include comments on every line of code Avoid: [advanced topic to exclude]

Template 2 — Debugging

Task: Find the error in this [C / C++ / Python] program Context: The program should [describe expected behaviour]. Instead it [describe actual wrong behaviour / error message]. The problem happens when I input: [test case] Output: 1. Identify the error and its exact location 2. Explain why it happens step by step 3. Show the corrected code 4. Tell me how to prevent this type of error in future Code: [paste your code here]

Template 3 — Code Generation

Role: Act as a [C / C++ / Python] programming tutor Task: Write a [language] program that [specific task description] Context: I am a beginner. The program should handle [input description] and produce [output description] Output: Add a comment on every line. Show expected output for [test case] Avoid: [advanced features / libraries not yet covered]

Pro Tip: Keep a Prompt Library

Save your best prompts in a text file. When you find a prompt that works well — keep it and reuse it. Adjust the [variables] for each new task. Over time you will build a personal toolkit that saves you hours.

Iterative Refinement

The most powerful way to use AI — as a real conversation, not a single question.

The Concept

Your first prompt rarely produces the perfect answer. That is completely normal — and expected. The real skill is knowing how to refine, extend, and improve across multiple turns of conversation.


AI remembers the context of your conversation. Each follow-up can build on what it already gave you.

Example — Building a C Program Step by Step

Turn 1 — Start simple: "Write a C function that calculates the average of an array of integers." Turn 2 — Add a specific requirement: "Good. Now add input validation: the function should return -1 if the array is empty (size = 0). Do not change anything else." Turn 3 — Extend further: "Now also find and return both the minimum and maximum values from the same single pass through the array. Update the function signature to return a struct." Turn 4 — Ask for understanding: "Explain why using a single pass (one for loop) is more efficient than using separate loops for min, max, and average."

Refinement Phrases That Work

  • "Good. Now also add..." — extend the solution
  • "That is correct but I need it to handle..." — fix a gap
  • "Can you simplify line 8?" — improve a specific part
  • "Explain what the for loop on line 12 is doing." — understand before continuing
  • "Now optimise this for performance." — improve quality
  • "Rewrite this using a while loop instead." — learn alternatives

The Mindset Shift

Stop thinking of AI as a search engine where you type one question and get one answer. Think of it as a colleague sitting next to you. You work together, you ask follow-ups, you push back, you refine. That is when AI becomes truly powerful.

Common Mistakes When Using AI for Programming

Why students fail with AI — and the specific fix for each problem.

Mistake 1 — Vague Programming Prompts

Asking "write a sorting program" or "explain pointers" without any context produces textbook-level responses that are too generic to be useful for your specific learning needs.

Fix: Always specify the language, your level, the exact concept, and the format you need.

Mistake 2 — Blind Copy-Paste

Copying AI code into your program without understanding it means you have no idea if it is correct, if it matches your requirements, or why it works. If it breaks — you cannot fix it. If a teacher asks you to explain it — you cannot.

Fix: Read every line. If something is unclear — ask AI: "Explain line X in simple terms." Only submit code you can explain.

Mistake 3 — Not Verifying the Output

AI hallucinations happen in code too. A function can look correct, compile without warnings, and still produce wrong results — or use a library function that does not exist.

Fix: Compile the code. Run it with at least 2–3 different test inputs. Check the output manually against what you expect.

Mistake 4 — Asking for Too Much at Once

"Build me a complete student management system in C with file I/O, linked lists, and a menu" — this kind of prompt produces too much code to understand, with bugs you cannot locate.

Fix: Break it into pieces. Build one function at a time, understand it, test it, then ask for the next part. Use iterative refinement.

Mistake 5 — Not Using the Error Message

When code fails, students often describe the problem in vague terms. "My program does not work" gives AI very little to work with.

❌ "My C program does not work, please fix it" ✅ "My C program gives this compiler error: 'error: expected ';' before 'return' on line 15' Here is the code: [paste code]. Explain the error and fix it."

Real Developer Workflow with AI

This is how professional developers actually integrate AI into their work — not as a shortcut, but as a thinking partner.

The Complete Workflow

Problem
→
Write Prompt
→
AI Output
→
Understand
→
Test
→
Improve

Each Step Explained

  • Problem — Define what you need to solve, completely and specifically. The clearer your problem definition, the better your prompt will be.
  • Write Prompt — Use the template structure. Apply the principles. Check the 6-question checklist before sending.
  • AI Output — Read the complete response. Do not skim. Highlight anything you do not fully understand.
  • Understand — Before running any code — understand every line. Ask follow-up questions for anything unclear. This is the step most people skip — and it is the most important.
  • Test — Compile and run the code. Test with your expected input. Test with edge cases (empty, zero, large numbers, wrong type).
  • Improve — If it is not right — refine your prompt and go again. If it is right — ask AI: "How could this be written more efficiently?" or "What are the limitations of this approach?"

A Real Student Example

Problem: I need to write a C program to find duplicate numbers in an array Step 1 — Write Prompt: "Act as a C tutor. Write a function that finds and prints duplicate values in an integer array of size n. Add comments on each line. Show expected output for: {1, 2, 3, 2, 4, 1, 5}" Step 2 — Read and understand the output Step 3 — Compile and test Step 4 — Iterative improvement: "Good. Now modify the function to also count how many times each duplicate appears. Do not change the original logic." Step 5 — Test again, understand the new parts

The Loop Mindset

The workflow is a loop, not a straight line. Your first attempt might need 2–3 refinements before it is right. That is normal, that is the process, and that is how real developers work. The skill is in the iteration — not the first prompt.
