# Capstone Lab: Build a Mini Python Program
**AI for IT Students Workshop — Day 2**
Time: 45 minutes | Work solo or in pairs

---

## Overview

You will use AI as your coding assistant to build a small Python program from scratch. This lab brings together everything from both days: prompting skills (Day 1) and Python programming with AI help (Day 2).

---

## Choose a Project

Pick ONE project from the six options below. Start simple — a working program with 3–5 features is better than an unfinished complex one.

---

### Project 1: Grade Calculator
**What it does:** Takes a list of marks and calculates the average, highest, and lowest score.

**Starter prompt for AI:**
> "Write a Python program that takes a list of 5 student marks, then prints the average mark, the highest mark, and the lowest mark. Keep it under 15 lines."

**Extension (if time allows):** Ask the user to enter marks one by one using `input()`.

---

### Project 2: Simple Quiz Game
**What it does:** Asks the user 3 multiple-choice questions and shows their score at the end.

**Starter prompt for AI:**
> "Write a Python quiz game with 3 questions. Each question has 3 answer options (A, B, C). After all questions, show how many answers were correct. Keep it simple and under 25 lines."

**Extension:** Add a "well done" or "try again" message depending on the score.

---

### Project 3: Student Name List Manager
**What it does:** Lets the user add student names to a list, view the list, and count how many names are in it.

**Starter prompt for AI:**
> "Write a Python program with a menu: 1) Add a name, 2) Show all names, 3) Count names, 4) Quit. Use a list to store names. Keep it simple."

**Extension:** Let the user search for a name in the list.

---

### Project 4: Number Guessing Game
**What it does:** Picks a random number between 1 and 10. The user guesses until they get it right.

**Starter prompt for AI:**
> "Write a Python number guessing game. Pick a random number between 1 and 10. Ask the user to guess. Tell them if they are too high, too low, or correct. Count how many guesses they took."

**Extension:** Limit the user to 5 guesses.

---

### Project 5: File Score Reader
**What it does:** Reads numbers from `scores.txt` (provided) and prints the average.

**Starter prompt for AI:**
> "Write a Python program that opens a file called scores.txt, reads all the numbers (one per line), calculates the average, and prints it."

**Note:** Use the `scores.txt` file provided in the supplements folder.

**Extension:** Also print the highest and lowest score.

---

### Project 6: Simple Contact Book
**What it does:** Stores names and phone numbers in a dictionary. Let the user add contacts and look them up.

**Starter prompt for AI:**
> "Write a Python program that acts as a contact book using a dictionary. The user can: 1) Add a name and phone number, 2) Look up a phone number by name, 3) Quit. Keep it simple."

**Extension:** Tell the user if a name is not found.

---

## How to Work

1. **Prompt AI** using the starter prompt — then adjust based on what it gives you
2. **Run the code** on Replit
3. **Fix errors** by copying error messages back into AI
4. **Test it** — make sure all features work
5. **Show your instructor** before the 45 minutes is up

---

## Deliverable

A working Python program you can demo live. Be ready to explain:
1. What your program does
2. One prompt you wrote to AI and what it gave you
3. One thing you had to fix or improve

---

## Grading Checklist

| Criteria | Points |
|---|---|
| Program runs without errors | 40 |
| At least 2 features work correctly | 30 |
| Can explain what the code does | 20 |
| Used AI effectively (asked good prompts) | 10 |
| **Total** | **100** |

---

## Tips
- Start with the starter prompt, then add one feature at a time
- If AI gives you too much code, say: *"Simplify it — keep only the core features"*
- If you hit an error, paste it into AI: *"I got this error: [paste error]. What does it mean?"*
- A simple working program beats an impressive broken one
